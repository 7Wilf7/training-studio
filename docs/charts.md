# 图表

图表子视图（Training tab → **Charts**）按你选的时间窗口渲染图表。所有图都受 Training tab 顶部 **All activities ▼** 筛选器的影响。

> 默认时间窗口是**最近 4 周**。

## 时间窗口

图表上方 5 个 preset：

- 最近 **4 周** / 最近 **8 周**
- 最近 **6 个月** / 最近 **12 个月**
- 最近 **5 年**

选一个 preset 同时改**所有三张图**的窗口。窗口永远以「现在」结尾。

> 这个时间窗口和 Activities 列表的时间标签（All Time / This Week / …）是**独立的**。改图表的窗口不会影响列表的统计范围。

## 1. 跑步距离趋势

折线图，按桶汇总**跑步距离（km）**，桶大小随 preset 是周/月/年。「跑步」指 Road Run、Trail Run、Hiking、Floor Climbing 四类的总和。Strength 和 HIIT 不进，因为没有意义的距离。

- 周桶按周一→周日，标签形如 `5-18~24` 或跨月 `5-30~6-5`。
- 月桶按自然月。
- 年桶按自然年。
- 线条下方的填充色纯装饰 —— 「等高线」视觉风格。

## 2. 路跑类型分布

水平条形图，按当前窗口内 Road Run 的时间分布到 4 个配速子类（Easy / Aerobic / Tempo / Interval）。

**按时长（秒）加权**，不是按次数。理由：一次 90 分钟 tempo 的训练负荷大于 3 次 20 分钟轻松跑，按时长加权更能反映强度分配。

Trail Run / Hiking / Floor Climbing 不进这个图 —— 它们没配速子类。Strength 和 HIIT 也不进。

## 力量训练时长按部位平摊

如果你**用筛选器选中了某个具体部位**（比如「Upper Body」），一次涉及多部位的力量训练会按比例分配时长。例如：一次「30 分钟，上肢 + 下肢」的训练，在「Upper Body」筛选下只贡献 15 分钟，避免把整段 30 分钟全算到一个部位头上。

不筛选具体部位时（看「全部力量」），仍然是完整时长。

## 提示

- 顶部筛选器切了之后图会立刻重渲染，可以用来对比某一类训练在不同时间段的变化。
- 路跑类型条形色从深到浅渐变，呼应强度递增。
- 如果某张图空着，多半是当前窗口里没有对应数据 —— 试着拉长时间窗口（比如从 4 周改到 12 个月）。
