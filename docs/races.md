# 赛事管理

Races tab 管两件相关的事：你**计划参加**的未来赛事，和你**已经跑过**的历史赛事。它们共用同一个表单。顶部的 **PR 栏**会从历史里自动聚合每个类别的最佳成绩。

## 赛事类别

七个固定类别，按显示顺序：

1. **10K**
2. **Half Marathon（半马）**
3. **Marathon（全马）**
4. **Trail（越野）** —— 距离不定，有爬升字段
5. **Spartan（斯巴达）** —— 按 tier 分（Sprint / Super / Beast / Ultra），按难度排序
6. **Hyrox** —— 固定室内赛制，无距离/爬升字段
7. **Other（其他）** —— 兜底类别

这个顺序同时决定了表单下拉框和 PR 栏的展示顺序。

## 目标赛事

按**日期升序**排（最近的下一场在最前）。每条包含：

- **Priority（优先级）** —— A / B / C。A = "决定整个赛季的核心赛"，B = 次要，C = 参与性质。优先级会影响 PR 栏颜色，发给 AI 教练时会写明「Priority A/B/C」让教练知道哪场最重要。
- **类别** + （仅 Spartan）tier
- **日期**
- **距离 / 爬升** —— Trail 和 Other 有；路跑类（10K/HM/全马）距离已由类别决定所以隐藏；Hyrox 也隐藏
- **名称**

### 添加目标赛事

1. Races tab → **Add Target Race**。
2. **先选类别** —— 表单会根据类别动态显示/隐藏字段。
3. 填名称、日期、优先级，按需要填距离/爬升。
4. 如果日期填在过去，会弹警告问要不要直接归到历史里。

## 历史赛事

按**日期降序**排（最近的最前）。表单跟目标一样，多一个**完赛时间**字段（时:分:秒）。Trail 历史还能在 PR 栏上挂一个可选的 **ITRA Performance Index** 标记。

## 筛选

目标和历史各有一行类别筛选 chips。多选 —— 点几个就筛几个，点 **All** 清空。区块标题旁的计数器在有筛选时会显示 `已筛 / 总数`。

## 顶部 PR 栏

放在 Races tab 顶部，从历史列表里自动聚合每类的 PR：

- **10K / 半马 / 全马 / Hyrox / Other** —— 按**最快完赛时间**排
- **Trail** —— 按**最长距离**排（越野「PR」更看重耐力而非速度）
- **Spartan** —— 按**最难 tier** 排（Ultra > Beast > Super > Sprint）

每张 PR 卡片下面的「+ other finishes」可以展开看这一类的其他成绩。

### ITRA

Trail 那张 PR 卡的右上角有一个 **ITRA** 小标记。点它录入或更新你的全局 ITRA 表现指数。这是**每个用户一个值**，不是每场赛事一个。

## 编辑 / 删除

点任一行进入内联编辑。没分类的行上有个小下拉框，直接设类别不用打开完整表单。最右边的 × 删除（带确认）。

## 跟 AI 教练的关系

- **目标赛事全部**发给教练（教练需要知道你接下来要打哪些比赛）。
- **历史赛事经过筛选**发给教练：每类最近 3 场，加上 Trail 最长距离一场 + Spartan 最难 tier 一场作为锚点。完整规则见 [AI 教练](ai-coach.md)。
