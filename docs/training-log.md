# 训练记录

每条训练都归到三大类之一：**跑步类（Run）**、**力量（Strength）**、**HIIT**。这份记录是所有统计的源头 —— 顶部统计条、图表、AI 教练能看到的数据、PR 栏，都从这里读。

## 六种活动类型

| 大类 | 具体类型 | 记录哪些字段 |
|---|---|---|
| 跑步 | Road Run（路跑）| 距离、时长、配速、心率、累计爬升、步频、GAP、有氧 TE |
| 跑步 | Trail Run（越野跑）| 距离、时长、配速、心率、累计爬升、有氧 TE |
| 跑步 | Hiking（徒步）| 距离、时长、配速、心率、累计爬升、有氧 TE |
| 跑步 | Floor Climbing（爬楼/楼梯机）| 时长、心率、累计爬升（无水平距离）|
| 力量 | Strength | 时长、心率、可选身体部位 |
| HIIT | HIIT | 时长、心率、有氧 TE |

子类规则因大类而异：

- **Road Run** 必须选一种配速子类（Easy / Aerobic / Tempo / Interval），可加 Race 标记。详细规则见 [Running 配速分类](running.md)。
- **其他跑步类型**（Trail / Hiking / Floor Climbing）可加 Race 标记，但不分配速 —— 地形决定配速，按心率分类会误导。
- **Strength** 可勾选 Upper Body / Lower Body / Core 任意组合。
- **HIIT** 没有子类。

> 「积极恢复」（按摩、瑜伽放松等）**不是**一种训练类型。它是**日级标签**，在日历上某一天上勾选。

## 手机端卡片显示

手机端每张训练卡分两行。第二行是按类型定制的紧凑指标，**时长统一放最前面**，方便竖向扫读：

| 类型 | 紧凑指标顺序 |
|---|---|
| Road Run | 时长 · 距离 · 配速 |
| Trail Run / Hiking | 时长 · 距离 · 爬升 |
| Floor Climbing | 时长 · 爬升 |
| Strength / HIIT | 时长 · 心率 |

剩下的字段（步频、TE、GAP、最大心率…）点开卡片展开后才显示。

## 添加训练（手动）

1. 打开 Training，进入 Activities 子视图（手机和电脑都默认在这）。
2. 点 **Add Activity**（手机端是 **+ Add**，按钮在筛选条下方）。
3. 选日期和类型。Road Run 默认子类 Easy Run，其他类型默认空。
4. 按你测到的填。**日期和时长必填**，距离/心率/爬升/步频/GAP/TE 都可选 —— 没量到就留空。
5. **GAP 输入** —— 两个独立数字框（**分 | 秒**），中间是固定冒号，不用自己敲。秒只接受两位，离开输入框会自动补成 "06" 这种两位形式。
6. 「开始时间」可选 —— 留空就用保存这一刻的天气；填了就抓那个时间点的历史天气（最近 24 小时）或未来预报。
7. 保存。新行**立刻**出现在列表顶部，表单同时关闭 —— 天气抓取和云端写入都挪到了后台跑，不会再卡屏。如果云端写入失败，刚加的行会消失，并弹一条错误提示让你重试。

> 手机端的跑步子类 chip 用短文案（Easy / Aerobic / Tempo / Interval），节省横向空间；电脑端仍然是完整的 Easy Run / Aerobic Run / Tempo Run / Interval Run。

## 天气

每条手动添加的训练会自动附上当时的天气快照（温度、体感、湿度、天气状况、风速、AQI），显示在卡片上。

- **第一次添加时**会请求位置权限。授权后用手机/电脑的 GPS 定位，精确到街道。
- **拒绝授权 / 没装定位 / 海外 VPN 卡住**的情况：去个人资料 → 位置，点"用当前位置"或手填坐标兜底。
- **CSV 批量导入**和**日历计划**不抓天气（前者是历史数据，后者要看那天才知道）。
- **过去的训练**（开始时间填的是 24 小时内的某一刻）会抓那个时间点的历史天气。
- **未来的训练**（开始时间填的是接下来 7 天内）会抓那天的预报，保存进去。

天气抓不到时（位置拒绝、网络不通、彩云额度用完）静默跳过，记录正常保存，只是少了天气字段，不影响其它功能。

## 编辑训练

- **电脑**：点列表里任一行进入内联编辑。点外面取消 —— 有未保存改动会弹确认框。
- **手机**：点卡片展开 → 点 **Edit** 按钮。

## 批量操作

- **Add / Upload / Select / Sort** 现在集中在 Activities 顶部同一行，并配有图标，手机端也保持一行显示。
- **Select mode**：勾选多行批量删除。
- **Upload .csv**：打开 Garmin CSV 导入流程，详见 [数据导入](data-import.md)。
- **Sort**：按日期、距离、时长或心率排序当前列表。

## 筛选和时间范围

- **筛选** —— 顶部 **All activities ▼** 下拉框，同时作用于 Activities 列表和图表。可按跑步子类、力量身体部位、HIIT 筛选。
- **时间范围** —— **All Time / This Week / This Month / This Year / Custom** 标签，控制统计条和列表的范围。**图表有自己独立的时间范围**，详见 [图表](charts.md)。

## 关于计划训练

从 AI 教练那里一键导入到日历的**计划训练**只在日历上以虚线框显示。它们**不**进 Training 列表，**不**进统计条，**不**进图表，**不**进 PR 栏。只有等你手动标记完成后才会计入。
