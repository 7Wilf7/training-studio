# Running 配速分类

路跑（Road Run）每条记录都会带一个**配速子类**，让图表和 AI 教练能分析你的强度分布。一共 4 种配速子类 + 1 个独立标记。

## 子类含义

| 子类 | 含义 |
|---|---|
| Easy Run | 对话配速，恢复跑 / 打底有氧 |
| Aerobic Run | 稳定有氧，乳酸阈以下 |
| Tempo Run | 乳酸阈区间，「舒适地难」 |
| Interval Run | 阈值以上的间歇，VO2max 区间 |

外加一个可与任何配速子类共存的独立标记：

- **Race（比赛）** —— 标记为比赛性质的训练。在活动列表里 chip 颜色会变成警示橙。

## 个性化推荐（取代了固定阈值）

只要你在个人资料里填好了 **Resting HR、Max HR 和心率区间方法**，系统会按你的个性化心率区间帮你**推荐**配速子类：

| 平均心率落点 | 推荐 |
|---|---|
| 低于 Z2 下沿 | Easy Run |
| Z2 下半段 | Easy Run |
| Z2 上半段 | Aerobic Run |
| Z3 | Tempo Run |
| Z4 及以上 | 不推荐（让你自己选） |

> Z4+ 不主动猜 —— 因为靠平均心率分不清是 Tempo / 阈值 / Interval。间歇训练必须手动选。

**两个地方会用到这个推荐：**

1. **手动录入训练时**：你在表单里填好平均心率后，匹配的那个配速 chip 会出现「推荐」标记和一个绿色描边，下方还有一行说明文字。**它只是建议** —— 你想选别的就直接点别的 chip，系统不会替你做主。
2. **Garmin CSV 导入时**：每条 Road Run 行会按上面的规则自动预填一个配速子类。导入预览界面里仍然可以逐条改。

## 没填心率信息时的兜底

如果你还没填 Resting HR / Max HR，系统不知道你的个性化区间，会退回旧的统一阈值（低于 150 → Easy / 150–164 → Aerobic / 165–174 → Tempo / 175+ 不推荐）。**强烈建议**去填一下个人资料里的两个心率，这样推荐才靠谱。

> Trail Run / Hiking / Floor Climbing **不**自动分类 —— 这些活动地形决定配速，按心率分容易判错。

## 手动选择子类

新建或编辑活动时，Road Run 默认是 **Easy Run**。保存前点其他配速 chip 即可换。配速 chips 是互斥的（选新的替换旧的）；Race 标记独立切换。

电脑端和手机端的子类选择 UI 一致，按钮位置都在表单的活动类型下方。

## 体现在哪里

- 在 **Training → Charts → 路跑类型分布** 里，按子类聚合 Road Run 训练，权重是**时长（秒）**而非次数。一次 90 分钟 tempo 的训练负荷大于 3 次 20 分钟轻松跑，按时长加权更能反映强度分配。
- 其他跑步类型不进这个图（没配速子类）。
- 发给 AI 教练的近期训练数据里，每条活动后面都带子类，教练能区分上周 60 km 是轻松底子还是 tempo 偏多。

## 提示

- 子类一旦选错可以随时回 Training 列表点开重选。
