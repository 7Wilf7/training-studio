# AI 教练

每日打卡式的对话教练，基于你的真实训练数据、目标赛事和长期记忆给个性化建议。

## 初次配置

1. 打开 API 设置（**电脑**：顶部右侧 **API key** 按钮；**手机**：Settings tab → API key）。
2. 选一个 **Provider**：
   - **DeepSeek** —— 在 [platform.deepseek.com](https://platform.deepseek.com/) 注册申请 Key。便宜、速度快。
   - **Claude API（第三方）** —— 在 [console.claudeapi.com](https://console.claudeapi.com/) 注册申请 Key。**不是** Anthropic 官方，是第三方中转服务，但接口和模型保持同步。质量更高、价格也更高。
3. 设置弹窗下方只会显示**当前 provider** 的设置；切到另一个 provider 时另一组 Key 仍然保留在后台，切回来还在。可以两边都填好做横评。
4. **接入线路（仅 Claude API）**：第三方服务提供多条线路（默认 / 香港 / 日本 / 新加坡），按当前网络延迟选一条。这个选择按设备保存 —— 手机和电脑可以走不同线路。
5. **模型**：每个 provider 都**锁定为该家当前最强模型**（DeepSeek = V4 Pro，Claude = Opus 4.7），用户不需要选。厂商有新旗舰发布，应用下次启动会自动跟进。
6. 打开 **AI Coach** tab。对话历史会自动保存，刷新和换设备都不会丢。

> 切换 Provider 只影响下一条消息发给谁；过往的对话历史保留不动。

## 教练每次能看到什么

每次你发消息，教练都会拿到这些信息：

- **个人资料** —— 年龄、性别、城市、职业、训练年限、伤病史、可用器材，以及（如果填了 Resting HR + Max HR）个人心率区间。
- **教练配置** —— 三个轴（风格、输出长度、风险提醒强度），可在 Coach Config 面板调。
- **长期记忆（Memory）** —— 你自己（或 AI 自动）维护的关于你的持久事实。
- **当前时间**和**全部目标赛事**。
- **最近一场目标赛事的比赛日天气**（仅当你给那场赛事填了地点；室内的 Hyrox 不算）—— 临近比赛（约两周内）是真实预报，还早则是该地点该日期的往年同期典型天气。教练据此给比赛日的配速 / 补水 / 装备建议、提前提示高温高湿。只带**最近这一场**，后面几个月的比赛不会一并念出来。
- **精选历史赛事**（每类最近 3 场 + Trail 最长 + Spartan 最难）。
- **最近 8 周的周训练量趋势** —— 每周一行：跑步类（Road Run / Trail Run / Hiking / Floor Climbing）的总距离、总爬升、训练次数。让教练能看出"这周比上周猛涨"这类周期化 / 伤病风险信号，而不只是孤立的最近 10 条。只统计跑步类训练；没有任何跑步数据的周（以及开头连续的空周）不显示。
- **最近 10 条已完成训练**（计划训练不包括在内）—— 含你填的 **RPE（自觉用力）**和**备注**，行尾还会附上当时的天气（如果有）。
- **最近 21 天的当日标记** —— 你在日历上勾的 按摩 / 拉伸 / 睡眠差 / 生病 / 出差。教练排计划、评估负荷时会参考（生病、出差尤其关键）。没勾过就不出现。
- **当前天气**（温度、体感、湿度、天气状况、风速、AQI）—— 实时拉取。
- **未来 7 天的天气预报**（每天一行：最高/最低、体感、湿度、天气状况、风速、AQI）—— 无论你那天有没有训练计划，预报都会发给教练，让它在排新计划时主动按天气挑日子。
- **接下来 7 天内的计划训练 + 每天的天气预报** —— 如果你日历上有计划，那条计划后面会再贴一次当天的预报，方便教练直接对照。
- **完整对话历史**。

> 教练不会主动获取你浏览器之外的任何数据。你给它什么，它就用什么。

### 关于训练数据的新鲜度

每次发消息**之前**都会重新从数据库拉一次最新训练记录，所以无论你是刚在同一个窗口里加的训练，还是在另一台设备上录的训练，下一条 Coach 回复都能看到。

加完一条训练立刻问 "我刚跑完帮我看看"，Coach 应该直接引用那条的距离/时长/心率 —— 如果它说"我看不到你最新的数据"或者"系统还没同步"，那是 LLM 在偷懒**幻觉**，把它当作错答处理，直接反问 "5/24 的数据应该有，请再看一下"，它就会承认。

## Memory（长期记忆）

一段你（或 AI）随时间维护的自由文本。这是「持久层」—— 比如：

- 「轻松日必须真的轻松，不然跟腱会犯」
- 「备战 2026 UTMB CCC，8 月底 A 级比赛」
- 「不喜欢被人在恢复跑里提速度」

两种更新方式：

- **Edit** —— 打开 Memory，点 Edit，自己写，Save。
- **Auto-update from chat** —— 把当前 memory + 最近对话发给 AI，让它产出一份建议更新版。你审核后选 Accept 或 Discard。建议内容按你的语言设置显示（面板里也能中/EN 切换）。

AI 在生成建议更新时被要求**只保留持久事实**，丢掉一次性琐事，控制在大约 500 字以内。

更新过程在**后台进行**：点了更新后即使切到别的页、关掉记忆面板也没关系，它会继续跑（更新中按钮有转圈动画）。跑完 App **顶部会弹出一条通知**，点一下就回到记忆面板，审核要保留 / 删除的条目。

## 长对话软提示

聊累计到 **40 条**消息时，对话上方会出一条琥珀色软提示，建议你把要点固化进 Memory + 清空对话；提示里直接有**"清空聊天"**按钮，点它清空前会再提醒你先更新一下记忆。原因是历史对话越长，旧消息越跟系统提示词争抢 AI 的注意力；定期固化进 Memory 能保持回复质量。

点提示上的按钮直接跳到 Memory 区域。

## 把建议导入到日历

每条 AI 回复下方都有一个 **📅 Import to Calendar** 按钮（默认显示，可在设置里关掉）。点它会让 AI 把那条回复里的训练建议解析成结构化的计划项目，弹一个 review modal 让你逐条调整后再导入。

- **休息日会空着**：只有教练明确安排了训练的那天才会生成计划项，没安排的日子留空（不会强行每天塞一个任务）；只有教练明确给了恢复性训练才会标成 Recovery。
- **时段**：教练若说了上午 / 下午，会带上，导入弹窗里也能改。
- **力量部位**：教练若指明上肢 / 下肢 / 核心，会一并带上，方便日历上看清楚练哪里；导入弹窗里可勾选调整。

导入的项目在日历上以**虚线框**显示，**不**计入统计或 PR 栏，除非你手动标记完成。

> 不走 AI 导入也行：日历某天点 **Plan a workout** 手动排，也能选时段（上午/下午）和力量部位。

> 不想看到这个按钮？在 AI Coach 设置里把「Calendar Import Button」切到 **Hide** 就行。关掉后所有回复下面都不再显示这个按钮 —— 想用时随时回来打开。

## Coach Config / Memory / Prompt 预览

电脑端这三块在 AI Coach tab 的顶部按钮区展开；手机端则点输入框旁边的 **⚙** 进入设置子页打开。

对话框上方会显示一排状态 chip：当前 Provider、教练模式、Memory 是否已设置、日历导入按钮是否显示。切换设置后这里会同步更新，方便发消息前快速确认当前教练状态。

### Coach Config（教练行为）

三个维度各三档：

- **风格**：温和鼓励 / 平衡理性 / 严格数据驱动
- **输出长度**：极简 / 标准 / 详细
- **风险提醒强度**：轻提醒 / 标准 / 严格监督

随时可改，下一条消息开始生效。

### Preview Prompt

可以看到具体发给 AI 的完整内容。预览有 EN / 中 切换 —— **英文版才是真正发给 AI 的**（英文指令模型执行更稳定），中文仅供你阅读理解。

如果你怀疑 Coach 看不到某条训练，**先打开 Prompt 预览扫一遍 [Recent Activities] 块**，里面有就是 Coach 在装睡，没有才是真同步问题。

## 提示

- API Key 只存在你自己账号下，别人看不到。
- 如果对话框显示 "No response."，多半是 API Key 不对、模型名不对、或者当前 Provider 那一栏的 Key 还没填。
- API 网络错误显示为对话里的临时气泡，**不**写入历史。
- **安卓 APP** 在等回复时切到后台再回来不会再被打断 —— 对话请求走的是系统原生网络层，不依赖 APP 的前台状态。
- **网页版**（特别是 iOS Safari）切走仍可能让请求中断；回来后点用户消息下方的 **↻ 重新发送** 即可重试。
