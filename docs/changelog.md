# 更新日志

> Training Studio 的"装修日志" —— 从 2026-05-15 第一根钢筋开始，到今天每一天加了什么、改了什么、拆了什么。

## 2026-05-29（周五晚）—— v0.3.1：开屏 logo 收拾干净

- 开屏「Training Studio」那一页的 logo 之前外面有一圈难看的深色窄边框。现在裁掉了，纹理铺到圆角边，**跟桌面图标长得一样**了。
- 安卓 12+ 开机时系统会先闪一下它自己的开屏页，原来是浅色卡片，跟后面的米色页不连贯。现在把系统开屏页背景也统一成**米色**，两段过渡更顺。

> 安卓 12+ 系统强制有一层自己的开屏，没法完全去掉，但现在前后都是米色 + 同一个 logo，看起来基本连贯。

## 2026-05-29（周五）—— 日历今日天气 / Training 顶部 / 标签可读性

**日历今日天气卡修复**

- 之前"今天"那张天气卡里温度和体感一直空着。现在显示**实时当前温度 + 体感 + 湿度 + 风 + AQI**（跟 AI Coach 顶部用的是同一份实时数据）。未来几天仍然是当天最高/最低。

**Training 顶部（手机端压缩）**

- 手机端原来筛选 / Activities-Charts 切换 / 时间范围是**三行竖排**，太占高度。现在压成**两行**：第一行类型筛选 + Activities/Charts 切换并排，第二行时间范围。
- 类型筛选从"All activities"改名为 **All Types / 全部类型**（它本来就是按运动类型筛的，紧挨着 Activities 切换容易混）。
- 电脑端布局不变（空间够）。

**可读性**

- 表单里的**长标签不再全大写**（比如个人资料里的"Race Types You've Done"这种整句），改成正常大小写，好读多了。活动类型 tag、星期、组别标签这些短标记仍然保留大写。

## 2026-05-29（周五）—— v0.3.0：APP 图标 / 开屏 / 返回键

**APP 图标（安卓）**

- 图标外面那一圈纯色背景去掉了，现在跟 PWA 一样满幅铺到边，是干净的整块 logo。

**开屏**

- 原来开机要先闪一个 logo、再闪一个写着"Training Studio LOADING…"的页面，两段式。现在合并成**一个页面**：居中 logo + 下方"Training Studio"，加载完直接进 App。
- 开屏 logo 不再有那一圈淡淡的边线，是干净的圆角整块。

**返回键 / 后台常驻（安卓）**

- 以前按返回键 / 侧滑返回会**直接退回桌面**，再进就得重新开屏（像被杀掉重开）。现在有了正常的返回逻辑：
  - 有弹窗开着 → 返回**先关弹窗**（比如改密码、设置、各种弹窗）
  - 没有弹窗但不在 Training 页 → 返回**回到 Training 页**
  - 已经在 Training 页 → 返回**退到后台**（像按 Home 键），App 留在后台不被杀，再点图标**秒回**原来的状态，不再重新开屏。

> 这一版需要更新 APK 才能体验（图标、开屏、返回键都是安卓原生层的改动，网页版不涉及）。

## 2026-05-28（周四晚）—— 体验回合：保存不卡 / 设置重组 / 自带 token

继上午的天气接入之后，又集中处理了一批体验问题。

**保存 / 删除不再卡顿**

- 手动添加活动点保存后，表单**立刻关闭**，活动条立刻出现在列表里；天气抓取和云端写入都挪到后台跑。原来从点保存到表单关闭大约要等 1–3 秒。
- 日历里删除计划训练、删除赛事、清空对话历史这些操作也都是**点完立即生效**，云端写入失败才会回滚 + 弹错。
- AI 教练导入计划到日历时不再等"导入成功"的弹窗 —— 计划已经显示在日历上了。

**AI 教练后台切换不再失败**

- 之前在等回复时切走应用再回来会显示"network request failed"。安卓 APP 现在走系统原生网络层发请求，把 APP 切到后台不会中断对话。
- 网页版的限制还在（特别是 iOS Safari），切走可能仍会断；输入框下面的"重新发送"按钮可用作兜底。

**改密码加了校验 + 改成弹窗**

- 修改密码现在需要**先输入当前密码**，再输入两次新密码。当前密码不对会提示"密码不正确"并显示"忘记当前密码请联系管理员"。
- 改密码不再是整页，而是一个**居中弹窗 + 背景模糊**，关掉弹窗就回到原来的设置页。
- 手机端的"退出登录"从设置页底部移到了**邮箱二级菜单**里（跟改密码同级）。点邮箱条目展开，看到"修改密码"和"退出登录"两个选项。
- 桌面端右上角的邮箱下拉也同步统一：原来的单独"Sign out"按钮合并进了下拉菜单。

**设置页重新组织成三大块**

- 自上而下：**账号 / API / 其他**。
- 账号：显示名 + 邮箱（展开二级菜单）+ 个人资料。
- **API：AI 教练 API + 彩云天气 API**（详见下条）。
- 其他：语言 / 默认位置 / 使用手册 / 应用版本（含检查更新）。

**彩云天气 API 自带 Token**

- 设置 → API → 彩云天气 API 新增 token 输入框，可以填自己在彩云官网申请的开发者 token。
- 填了之后，所有天气请求都走你自己的 token + 配额；留空就继续走应用的共享 token（有限流）。
- 桌面端右上角新增一个☁ 图标按钮直达这个设置。
- 申请入口：dashboard.caiyunapp.com（免费）。

**日历改版：7 天天气卡片在下方**

- 日历格子里的小天气图标和温度区间**删掉了**，格子重新干净起来。
- 下方新增一条 **7 张天气卡**（今天 + 未来 6 天），每张显示日期、天气图标、最高/最低温、体感、湿度、风速、AQI。
- 手机端**日历部分会固定在屏幕顶部**，下方可以滚动浏览 7 天天气卡，不会失去对日历的视觉锚定。

**AI 教练 Prompt 改成全量 7 天天气**

- 之前 prompt 里"未来天气"只在你已经安排了训练计划的那一天才会塞进去。现在改成**未来 7 天的天气一次性全发给教练**，让它在排新计划的时候就主动参考天气挑日子。
- 如果某一天你已经有计划，预报仍然继续附在那条计划后面。

**Race 文案统一 + 斯巴达上官方色**

- "Half Marathon" 改成 **Half Mar.**，"Marathon" 改成 **Full Mar.**；中文是 **半马 / 全马**。
- 斯巴达 tier 用上官方颜色：**Sprint 红 / Super 蓝 / Beast 绿 / Ultra 黑**。表单里选 tier 时显示色块预览，比赛卡片上的 tier 标签也会带上对应颜色。

**录入活动小优化**

- 手机端添加活动时，跑步类型 chip 从"Easy Run / Aerobic Run / Tempo Run / Interval Run"缩成 **Easy / Aerobic / Tempo / Interval**（活动卡片上的紧凑视图同步缩短）。桌面端文案不变。
- GAP 输入框改成 **分 + 冒号 + 秒** 两个独立数字输入框，不用自己敲冒号，秒数自动补零（"6:30"）。

**手机底部 tab 放大**

- 5 个 tab 的图标和文字都调大了一档，更好按。

## 2026-05-28（周四）—— 天气接入

**手动添加训练自动抓天气快照**

- 添加活动时自动抓取当时的天气（温度、体感、湿度、天气状况、风速、AQI），保存进每条训练记录里。
- 表单新增**"开始时间"**字段（可选）：留空就抓保存这一刻的天气；填了就抓那个时间点的历史天气（最近 24 小时）或未来预报。
- 活动卡片紧凑视图行尾显示天气图标 + 温度 + 体感（差 ≥ 2°C 时）。展开后显示完整数据（湿度、风速、AQI）。

**日历**

- 日历每个格子（今天 + 未来 7 天）右上角显示天气图标和温度。未来用预报（最高/最低），今天用实时。
- 点开任意日期的弹窗，最上方有当日天气总结。**过去的日期**显示该天第一条训练录到的天气；**今天和未来**显示实时/预报。

**AI 教练**

- 教练 prompt 新增三块天气数据：当前实时天气、最近 10 条训练每条行尾附当时天气、未来 7 天内的计划训练加预报。
- 教练现在可以基于"今天 32°C 体感 38°C"这种实际数据建议配速和补给。

**默认位置设置**

- 设置页新增**"默认位置"**入口（手机：Settings tab；电脑：顶部右侧📍按钮）。
- 当设备无法提供 GPS（APK 未授权、浏览器拒绝）时，用默认坐标兜底。可以一键"从当前设备获取"。

**Android**

- 安装 `@capacitor/geolocation` 插件，APK 首次抓天气时会弹位置权限请求。
- AndroidManifest 加 `ACCESS_FINE_LOCATION` + `ACCESS_COARSE_LOCATION`。

**小修与发布**

- Training 列表右上角的排序菜单在手机上更紧凑，Distance / Duration 这类选项不会再把下拉框挤出屏幕。
- 准备 v0.2.0 更新包，手机端可通过 APP 内"检查更新"获取新版。
- 修复 Android APP 内天气加载失败：Training 添加记录、日历天气和 AI Coach 顶部天气会正常使用线上天气服务。

> 数据源是**彩云天气** REST API。Token 放在 Vercel 服务端，浏览器看不到。

## 2026-05-27（周三）—— 大杂烩：AI 卡顿修复 + Claude 接入 + 个性化跑步推荐

**UI 操作区打磨**

- 手机底部导航从纯文字改成图标 + 文字，Training / Calendar / Races / AI Coach / Settings 更容易扫。
- 电脑端顶部工具条和主 tab 也同步加上图标，Guide / 语言 / API / 个人资料入口更容易识别。
- Training 的 Add / Upload / Select / Sort 操作集中成带图标的一行，手机端更紧凑。
- AI Coach 对话框上方新增状态 chip，直接显示当前 Provider、教练模式、Memory 状态和日历导入按钮状态。

**AI Coach 回复支持 Markdown 渲染**

- 教练常用的格式（表格、`**粗体**`、`### 标题`、列表、`` `代码` ``、引用块）以前全是字面字符显示，读起来费劲。现在统一用 markdown 引擎渲染。
- 周训练计划那种「7 列大表格」会真正显示成表格；移动端表格如果太宽，可以在气泡内左右**横滑**查看。
- 长 URL / 模型 ID 这类不可断词的内容会自动换行，气泡不会被撑爆。

**AI Coach 体验**

- **发消息不再有 1-2 秒卡顿** —— 用户气泡现在按下发送就出现，不用等数据库写完。
- **每条用户消息下加了「↻ 重新发送」按钮**（只对最近一条显示）。切 app 回来发现报「网络请求失败」时一键重试。
- **网络错误文案改写**，明确告诉你切 app 可能让浏览器暂停了请求 —— 不再让你打开开发者工具看 Network 面板了。
- **移动端聊天气泡变宽**（85% → 94%），长消息不再挤在窄柱里。
- **日历导入按钮可隐藏** —— 在 AI Coach 设置里新增独立「Calendar Import Button」开关（默认显示），不想用就关掉，所有回复下面就不再出现这个按钮。开启时按钮以文字形式显示在每条教练回复下方。

**Claude API 回来了（第三方中转）+ 模型锁定为最强**

- API 设置加了 **Provider 切换器**：DeepSeek 和 Claude 都能用，两个 Key 同时存着，随时切换不需要重新粘。设置弹窗只显示当前 provider 的设置，UI 不再混在一起。
- Claude API 走的是**第三方中转服务**（console.claudeapi.com），不是 Anthropic 官方。提供 4 条接入线路（默认 / 香港 / 日本 / 新加坡），按当前网络选一条，按设备保存。
- **模型锁定为该 provider 的最强款**（DeepSeek = v4-pro，Claude = opus-4-7）。用户不再需要选 —— 厂商发布更强模型时由开发者统一更新，应用下次启动自动跟进。
- 设置弹窗顶部增加 **价格对比卡片** —— 列出两家的输入/输出价格、单条消息估算成本、倍数差。Guide 文档同步加了价格表。可以一眼看出 Claude 单条约 ¥0.20、DeepSeek 不到 2 分钱。
- 切换 Provider 只影响下一条消息发给谁，过去的对话历史保留。

**跑步类型推荐：从固定阈值 → 个性化心率区间**

- 录入训练时，填好平均心率后，系统会按你**个人的 5 区心率**推荐配速子类（Easy / Aerobic / Tempo），匹配的 chip 出现「推荐」标记 + 绿色描边。
- 推荐只是建议，你想选别的就直接点别的，系统不会替你做决定。
- Z4+ 不主动推荐 —— 平均心率分不清是 Tempo 还是 Interval，交给你手动选。
- 没填 Resting HR / Max HR 的用户会退回到旧的统一阈值。
- 同样的逻辑也用于 Garmin CSV 导入的自动预填。

**Training stats 重排 + 时间显示**

- 顶部统计卡片从 **Sessions / Distance / Ascent / Avg HR** 改成 **Sessions / Time / Distance / Ascent**。平均心率作为汇总指标参考价值不大，换成总时长更有用。
- 移动端的 Time 只显示「Xh」/「Xm」格式，避免换行。

**Charts 简化**

- 默认时间窗口从 8 周改为 4 周。
- 删除「心率区间分布图」—— 单凭每次的平均心率分桶低估了多样性，留着反而误导。
- 力量训练时长按部位**比例分配**：选中「Upper Body」筛选时，一次「上肢+下肢」30 分钟的训练在该图里只计 15 分钟。
