# 更新日志

> Training Studio 的"装修日志" —— 从 2026-05-15 第一根钢筋开始，到今天每一天加了什么、改了什么、拆了什么。

## 2026-05-30（周六）—— 赛事当地天气 + 卡片 / PR 排版微调

### 赛事当地天气（新）

- 添加 / 编辑赛事时可以填**比赛地点**：输入城市名（如"广州"）从下拉里选一个即可，精确到城市就够。Hyrox 是室内赛事，不显示这一项。
- 填了地点的**目标赛事**会显示比赛当天的天气：
  - **临近比赛（约两周内）**：显示当天**真实预报**（温度区间 + 体感）。
  - **还早（几个月后）**：显示该地点该日期的**往年同期典型天气**（多年平均的温度区间 + 体感 + 大致降水），方便提前定补给和装备——因为再准的预报也报不了几个月后。
- **AI 教练**也会看到**最近一场**目标赛事的比赛日天气（临近=预报，还早=往年常态），据此给比赛日的配速 / 补水 / 装备建议、提前提示高温高湿风险。只带最近那一场，不会把后面几个月的比赛天气都念一遍。室内的 Hyrox 不计入。

### 排版微调

- **PR（个人最好成绩）卡片的 ITRA 分**：之前钉在右上角，换字体后会盖住右侧的比赛成绩。现在挪到卡片**右下角**，跟"+N other finishes"同一行，不再遮挡。
- **训练卡 RPE / 天气顺序**：RPE 移到那行的**末尾**（天气排在它前面）。
- **训练卡展开后排版**：展开时各项数据尽量留在同一行，**天气单独换到下一行**，窄屏手机上不再把某个数据（如 aerobic TE）挤到第二行。
- **展开后数据更紧凑**：各项数据之间间距收紧并左右铺开，窄屏上尽量挤进一行（数据特别多的路跑在很窄的屏上仍可能换一项）。
- **力量训练子项显示**：折叠卡上"Lower Body / Upper Body"显示为"Lower / Upper"，避免和 Core 一起时把 Core 挤掉（中文不受影响）。
- **目标赛事倒计时**：小标挪到**赛事类别后面**；格式改成"还有 X 个月 Y 天"，进入一个月内才显示"X 周 Y 天 / X 天"。
- **每日推送开关**：开关按钮之前被撑得过大，现在恢复成正常细长的开关。
- **PR 卡片高度一致**：缩小 ITRA 小标的高度，越野赛那张 PR 卡不再比其他类别高一截。

### 修复

- **切换 App 回来不再频繁弹"位置精确度"系统弹窗**：之前每次 App 回到前台都会重新定位（高精度 GPS），触发安卓的位置精确度提示。现在只要天气数据还新鲜就直接用缓存、完全不调 GPS；确实需要重新拉取时也改用低精度（城市级足够），不再弹那个框。

### 收件箱 / 赛事 / 长训练天气

- **收件箱秒开 + 未读数及时清零**：收件箱内容在 App 启动时就加载好，点信封图标**立即显示**（不再每次现拉 1–2 秒）；已读后图标上的未读小红数字**马上消失**（之前会卡在"1"）。
- **收件箱里加了"去设置每日推送"按钮**：点了会跳到设置页并让"Daily coach push"那条**闪一下**，提示你在哪里设置。
- **目标赛事到比赛日 → 一键录成绩转历史**：到了比赛日，目标赛事卡上会出现"录入成绩"按钮，填完赛时间确认后这场就自动从目标移到历史。
- **日历当日标记排成两行**（之前三行）。
- **长训练（≥3 小时）记录全程天气**：像越野超马这种，只记起跑温度参考意义不大。现在录入这类训练会**采样全程多个时间点的天气**（起跑 / 每约 2 小时 / 结束），喂给 AI 教练，让它知道"起跑 15℃、午间 32℃"这种变化。训练卡上仍只显示起跑时的天气，保持清爽。
- **出差/旅行可填目的地**：在日历里勾"出差"标记后，会多出一个输入框填**去哪**（如"成都"）。这个目的地会喂给 AI 教练（给当地跑步建议、考虑气候/时差），每日推送也会引用（比如出发前祝你玩得开心、推荐当地能跑的地方或当地小吃）。

### 再一批微调 & 修复

- **展开数据自适应只对路跑生效**：之前误把所有类型都做成"铺开自适应"，越野跑等只有三两个数据时被拉得很开很怪。现在只有路跑（数据多）才收紧铺开，其他类型恢复正常左对齐。
- **当日标记可以标在未来日期了**：之前只能标当天和过去；现在未来日期也能标（比如提前标好出差日）。未来日期不显示"睡眠差"（还没发生）。
- **"睡眠差"标记不再占两行**：现在"睡眠差(昨晚)"在一行显示，排版变成第一行 按摩/拉伸/生病、第二行 睡眠差/出差。
- **修每日推送收不到手机通知**：之前缺少安卓通知渠道，系统会把推送悄悄丢掉（应用内收件箱仍能收到，所以看着像"通了但没提醒"）。现在补上了通知渠道并提高推送优先级，通知能正常弹出。

### Prompt 预览 / 记忆 / 赛事天气

- **赛事当天天气切过去秒显**：之前每次切到 Races 都要等 1–2 秒重新拉天气。现在做了缓存（往年同期数据基本不变，缓存更久；临近预报缓存到当天），不再每次都调接口。
- **Prompt 预览改成只显示"架构"**：预览里不再展示完整的提示词内容——核心教练指令隐藏，调用的你的数据（资料 / 目标赛事 / 训练记录 / 天气等）只列出**占位符 + 一句说明**，让你知道发了哪些类别，但不暴露具体内容和写死的提示词。（实际发给 AI 的内容不变。）
- **记忆更新改成逐条勾选**：点"从对话更新记忆"后，建议内容会**分条列出**、每条带勾选框（新增的标 NEW），你取消不想要的、确认后只保存勾上的，不用再整段读完盲接受。
- **教练设置菜单小排版**：PROMPT 那行把说明收进括号；"预览完整 Prompt"跟下面几项对齐。
- **教练记忆支持中英文双语**：从对话更新记忆时，AI 会**同时生成中文和英文两版**并各自存好；记忆界面上有 **EN / 中** 切换按钮，想看哪个看哪个。发给 AI 的固定是英文版。逐条勾选保留时，两种语言会同步保留对应的条目。
- **每日推送支持半点**：推送时间从只能选整点，扩展到**每半小时一档**（08:00 / 08:30 / 09:00…）。设置里改成下拉选择（手机上是滚动选择），一天仍最多 3 个时间。仅安卓 APK 实际推送。

## 2026-05-29（周五）—— 训练趋势 + RPE/备注 + 倒计时 + 每日推送 + 收件箱（v0.4.0）

### AI 教练看得到"周训练量趋势"了

- 之前教练只能看到孤立的最近 10 条训练，看不出周维度的量。现在每次发消息会多带一段**最近 8 周的周训练量**：每周一行，跑步类的总距离、总爬升、训练次数。教练因此能识别"这周比上周猛涨 40%，注意伤病"这类周期化 / 风险趋势。只统计跑步类；没跑步数据的周不显示。可在 Prompt 预览里看到这段。

### 每条训练加了 RPE + 备注（可选）

- 录入 / 编辑训练时多了两个**可选**字段：**RPE（自觉用力 1–10）**和**备注**。RPE 旁边有一句说明（1 = 很轻松，10 = 拼尽全力）。两者都会喂给 AI 教练 —— RPE 帮它估算训练负荷，备注（"换了新鞋""膝盖有点紧"）当作背景。所有活动类型都能填。

### 日历当日标记扩展

- 当日标记从只有"按摩"扩到 **按摩 / 拉伸 / 睡眠差 / 生病 / 出差**，一天可勾多个。这些也会喂给教练（最近 21 天内勾的），生病 / 出差对它排计划尤其关键。

### 每日教练推送（APK）

- 可设**最多 3 个**每日推送时间(适合一天多练),服务端按你的时区到点生成一条简短 AI 打卡推到手机。推送语言跟随 App 语言设置。仅安卓 APK 生效。

### 卡片 & 日历小改

- 训练卡**折叠状态**也显示 **RPE**(放在天气前);天气温度去掉小数,只显示整数,行更清爽。
- 日历当日标记改成**小图标**(💆按摩 / 🤸拉伸 / 😴睡眠差 / 🤒生病 / ✈️出差),手机端格子只显示图标(居中放在格子底部),多选叠多个;"睡眠差"标注成"(昨晚)"避免歧义。

### 目标赛事倒计时

- 每张未来的目标赛事卡上多了一个绿色小标：离比赛 7 天以上显示"还有 X 周"，最后一周显示"还有 X 天"，当天显示"就在今天"。配合训练周期安排堆量 / 减量。

### 应用内收件箱

- 收到的每日教练推送现在会**存进 App 里的"收件箱"**(入口在 **AI Coach 页右上角的信封图标**),系统通知划掉了也能回头再看。支持**点开标已读、单条删除、一键清空**;有未读时图标上带红色未读数。仅安卓 APK 会真正收到推送,所以收件箱里的内容也来自 APK。

### 全局字体焕新

- 整个 App 的字体换成更圆润、更有"运动手表"感的 **Outfit**(几何圆体),正文、界面、数字统一一套,数字靠等宽对齐保持整齐。中文不受影响(仍走系统字体)。

### 更新下载进度条

- 安卓 App 内"检查更新 → 下载安装"时,按钮下方多了一条**实时下载进度条**(显示百分比);服务器没给文件大小时退化成来回滚动的进度条,至少能看出在动。仅安卓 APK 生效。

### 顺手清理

- 清掉了若干历史的代码质量警告（不影响任何功能）。

### 修复

- **"从对话自动更新记忆"在安卓后台切换不再失败**：这个按钮之前漏走了系统原生网络层，APK 上点完切后台会复现"network request failed"。现在和发消息 / 导入日历一样走原生层。
- **刚加完的训练在"保存中"时暂时不能点进编辑**：新加的训练会先乐观显示、再后台写库。在写库完成前点进去编辑会写失败并回滚弹错。现在这种"保存中"的训练条会标出"保存中…"并暂时禁用编辑，写完即可正常编辑（通常 1–3 秒）。
- **安卓"检查更新"漏报"已是最新"**：之前从某个 beta 测试版检查更新时，会把预发布版误判成比正式版还新，于是即使有新正式版也提示"已是最新"。现在版本比较修正，预发布版正确排在正式版之前。

## 2026-05-29（周五）—— 大更新日：v0.3.0 → v0.3.3

一整天密集迭代，连发多个版本。下面按主题归类。

### 个人资料未保存提醒（v0.3.3）

- 在个人资料里改了东西但还没保存，这时按返回 / 点 ✕ / 点窗口外 / 点取消，会先**弹确认**「放弃未保存的修改？」，避免误触丢失。没改动就直接关，不打扰。

> 这一批（手册表格修复、AI 教练设置分组 + 手机底部弹出菜单、定位整合进个人资料、Training 间距 + 切换记忆、未保存提醒）打包进 **v0.3.3** APK。

### 保存 / 删除不再卡顿

- 手动添加活动点保存后，表单**立刻关闭**，活动条立刻出现在列表里；天气抓取和云端写入都挪到后台跑（原来要等 1–3 秒）。
- 日历删计划训练、删赛事、清空对话历史也都**点完立即生效**，云端写入失败才回滚 + 弹错。
- AI 教练导入计划到日历不再等"导入成功"弹窗 —— 计划已经显示在日历上了。

### AI 教练

- **后台切换不再失败**：之前等回复时切走 App 再回来会报"network request failed"。安卓 App 现在走系统原生网络层发请求，切后台不会中断对话。（网页版尤其 iOS Safari 仍可能断，输入框下方"重新发送"兜底。）
- **Prompt 改成全量 7 天天气**：之前只有你安排了计划的那天才带预报，现在未来 7 天的天气一次性全发给教练，让它排计划时主动按天气挑日子。
- **设置重新归类**：教练设置里把「编辑资料 / 教练配置 / 长期记忆」归到 **Prompt** 这一组下（它们都决定发给教练的内容），「日历导入按钮 / 清空对话」归到「对话」组。设置项图标统一（去掉了原来只有编辑资料有的那个小图标）。
- **手机端教练设置改成底部弹出菜单**，单手好够；之前是从页面顶部展开。

### 设置 & 账号

- **设置页重组成三大块**：账号 / API / 其他。账号区点邮箱展开二级菜单（改密码 / 退出登录）；API 区分 AI 教练 API + 彩云天气 API；其他放语言 / 默认位置 / 使用手册 / 应用版本。
- **改密码加校验 + 改成弹窗**：先输当前密码再输两次新密码；当前密码错会提示"不正确"+"忘记请联系管理员"。改成居中弹窗 + 背景模糊。桌面端"退出登录"也并进右上角邮箱下拉。
- **彩云天气自带 Token**：设置 → API → 彩云天气 API 可填自己在 dashboard.caiyunapp.com 申请的 token，走自己的配额；留空用共享 token。桌面右上角加了 ☁ 入口。

### 日历 & 天气

- **日历下方 7 天天气卡**：日历格子里的小天气图标/温度去掉，格子更干净；下方新增 7 张天气卡（今天 + 未来 6 天），固定高度两行（第一行星期·日期·图标·温度，第二行体感·湿度·风·AQI）。手机端日历固定在顶部，天气卡可下滑。
- **今日卡修复**：之前"今天"那张温度和体感一直空着，现在显示实时当前温度 + 体感 + 湿度 + 风 + AQI（跟 AI 教练顶部同一份实时数据）。
- **手机端日历格子高度缩小**一截，给天气区让位。
- **7 天天气加"更新于 HH:MM" + ↻ 实时刷新按钮**，点一下绕过缓存重抓。
- 新增[**天气**](weather.md)使用手册章节，讲清每种数据（实时 / 预报 / 训练快照）的更新时机、默认位置、自带 token。

### 赛事 & 录入

- **赛事文案统一**："Half Marathon" → **Half Mar.**，"Marathon" → **Full Mar.**；中文半马 / 全马。
- **斯巴达 tier 上官方色**：Sprint 红 / Super 蓝 / Beast 绿 / **Ultra 紫**。表单选 tier 有色块预览，卡片标签也带色。
- **GAP 输入拆成 分 + 冒号 + 秒** 两个数字框，不用自己敲冒号，秒自动补零。
- 手机端跑步类型 chip 缩短：Easy / Aerobic / Tempo / Interval（桌面端不变）。

### Training 顶部 & 可读性

- **手机端 Training 顶部从三行压成两行**：第一行类型筛选 + Activities/Charts 切换并排，第二行时间范围。电脑端不变。
- 类型筛选改名 **All Types / 全部类型**（本来就是按运动类型筛，紧挨 Activities 切换容易混）。
- **表单长标签不再全大写**（如个人资料里"Race Types You've Done"整句），改正常大小写；活动 tag / 星期 / 组别这些短标记仍大写。
- 手机底部 5 个 tab 的图标和文字调大一档。
- 手机端 Training 顶部时间周期行和统计卡之间的间距修匀（之前比别处宽一截）。

### 切换记忆

- 一次使用中的 sub-tab 选择会**记住**：比如在「比赛」看着"历史"，切去日历再切回来，仍停在"历史"；Training 的 Activities/Charts 选择同理。只有彻底关掉重开 App 才恢复默认。

### 定位整合进个人资料

- 以前定位分两处（个人资料里的"城市"文本 + 设置里的"默认位置"坐标），现在**统一到个人资料的「位置」**一个地方。设置页里的默认位置入口、桌面顶栏的 📍 按钮都去掉了。
- 个人资料 → 位置 点 **📍 用当前位置**：自动定位 + 按 App 语言**自动填好区/市级地址**（如"广州市天河区"）+ 保存坐标。用 GPS 的话一个字都不用打。也可手动输入地区名 + 经纬度兜底。
- 这一个位置同时喂**天气**（坐标）和 **AI 教练**（地址），不再两头维护。

### APP 图标 / 开屏 / 返回键（v0.3.0–v0.3.1，需更新 APK）

- **图标**：外面那圈纯色背景去掉，跟 PWA 一样满幅铺到边，干净整块。
- **开屏**：原来先闪一个 logo、再闪"Training Studio LOADING…"两段式，合并成**一页**（居中 logo + Training Studio）直接进 App。开屏 logo 也裁掉了外圈深色窄框，跟桌面图标一致。安卓 12+ 系统强制的那层开屏背景也统一成米色，过渡更顺。
- **返回键 / 后台常驻**：以前按返回 / 侧滑直接退回桌面、再进要重新开屏（像被杀）。现在：有弹窗先关弹窗 → 不在 Training 页就回 Training 页 → 已在 Training 页则退到后台（App 不被杀，再点秒回，不重开屏）。

### 应用内手册 + 应用内更新（v0.3.2，需更新 APK）

## 2026-05-27（周三）—— 大杂烩：AI 卡顿修复 + Claude 接入 + 个性化跑步推荐

**UI 操作区打磨**

- 手机底部导航从纯文字改成图标 + 文字，Training / Calendar / Races / AI Coach / Settings 更容易扫。
- 电脑端顶部工具条和主 tab 也同步加上图标，Guide / 语言 / API / 个人资料入口更容易识别。
- Training 的 Add / Upload / Select / Sort 操作集中成带图标的一行，手机端更紧凑。
- AI Coach 对话框上方新增状态 chip，直接显示当前 Provider、教练模式、Memory 状态和日历导入按钮状态。

**AI Coach 回复支持 Markdown 渲染**

- 教练常用的格式（表格、`**粗体**`、`### 标题`、列表、`` `代码` ``、引用块）以前全是字面字符显示，读起来费劲。现在统一用 markdown 引擎渲染。
- 周训练计划那种「7 列大表格」会真正显示成表格；移动端表格如果太宽，可以在气泡内左右**横滑**查看。
- 长 URL / 模型 ID 这类不可断词的内容会自动换行，气泡不会被撑爆。

**AI Coach 体验**

- **发消息不再有 1-2 秒卡顿** —— 用户气泡现在按下发送就出现，不用等数据库写完。
- **每条用户消息下加了「↻ 重新发送」按钮**（只对最近一条显示）。切 app 回来发现报「网络请求失败」时一键重试。
- **网络错误文案改写**，明确告诉你切 app 可能让浏览器暂停了请求 —— 不再让你打开开发者工具看 Network 面板了。
- **移动端聊天气泡变宽**（85% → 94%），长消息不再挤在窄柱里。
- **日历导入按钮可隐藏** —— 在 AI Coach 设置里新增独立「Calendar Import Button」开关（默认显示），不想用就关掉，所有回复下面就不再出现这个按钮。开启时按钮以文字形式显示在每条教练回复下方。

**Claude API 回来了（第三方中转）+ 模型锁定为最强**

- API 设置加了 **Provider 切换器**：DeepSeek 和 Claude 都能用，两个 Key 同时存着，随时切换不需要重新粘。设置弹窗只显示当前 provider 的设置，UI 不再混在一起。
- Claude API 走的是**第三方中转服务**（console.claudeapi.com），不是 Anthropic 官方。提供 4 条接入线路（默认 / 香港 / 日本 / 新加坡），按当前网络选一条，按设备保存。
- **模型锁定为该 provider 的最强款**（DeepSeek = v4-pro，Claude = opus-4-7）。用户不再需要选 —— 厂商发布更强模型时由开发者统一更新，应用下次启动自动跟进。
- 设置弹窗顶部增加 **价格对比卡片** —— 列出两家的输入/输出价格、单条消息估算成本、倍数差。Guide 文档同步加了价格表。可以一眼看出 Claude 单条约 ¥0.20、DeepSeek 不到 2 分钱。
- 切换 Provider 只影响下一条消息发给谁，过去的对话历史保留。

**跑步类型推荐：从固定阈值 → 个性化心率区间**

- 录入训练时，填好平均心率后，系统会按你**个人的 5 区心率**推荐配速子类（Easy / Aerobic / Tempo），匹配的 chip 出现「推荐」标记 + 绿色描边。
- 推荐只是建议，你想选别的就直接点别的，系统不会替你做决定。
- Z4+ 不主动推荐 —— 平均心率分不清是 Tempo 还是 Interval，交给你手动选。
- 没填 Resting HR / Max HR 的用户会退回到旧的统一阈值。
- 同样的逻辑也用于 Garmin CSV 导入的自动预填。

**Training stats 重排 + 时间显示**

- 顶部统计卡片从 **Sessions / Distance / Ascent / Avg HR** 改成 **Sessions / Time / Distance / Ascent**。平均心率作为汇总指标参考价值不大，换成总时长更有用。
- 移动端的 Time 只显示「Xh」/「Xm」格式，避免换行。

**Charts 简化**

- 默认时间窗口从 8 周改为 4 周。
- 删除「心率区间分布图」—— 单凭每次的平均心率分桶低估了多样性，留着反而误导。
- 力量训练时长按部位**比例分配**：选中「Upper Body」筛选时，一次「上肢+下肢」30 分钟的训练在该图里只计 15 分钟。
