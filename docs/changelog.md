# 更新日志

> Training Studio 的"装修日志" —— 从 2026-05-15 第一根钢筋开始，到今天每一天加了什么、改了什么、拆了什么。

## 2026-05-29（周五）—— 大更新日：v0.3.0 → v0.3.2

一整天密集迭代，连发三个版本。下面按主题归类。

### 保存 / 删除不再卡顿

- 手动添加活动点保存后，表单**立刻关闭**，活动条立刻出现在列表里；天气抓取和云端写入都挪到后台跑（原来要等 1–3 秒）。
- 日历删计划训练、删赛事、清空对话历史也都**点完立即生效**，云端写入失败才回滚 + 弹错。
- AI 教练导入计划到日历不再等"导入成功"弹窗 —— 计划已经显示在日历上了。

### AI 教练

- **后台切换不再失败**：之前等回复时切走 App 再回来会报"network request failed"。安卓 App 现在走系统原生网络层发请求，切后台不会中断对话。（网页版尤其 iOS Safari 仍可能断，输入框下方"重新发送"兜底。）
- **Prompt 改成全量 7 天天气**：之前只有你安排了计划的那天才带预报，现在未来 7 天的天气一次性全发给教练，让它排计划时主动按天气挑日子。
- **设置重新归类**：教练设置里把「编辑资料 / 教练配置 / 长期记忆」归到 **Prompt** 这一组下（它们都决定发给教练的内容），「日历导入按钮 / 清空对话」归到「对话」组。设置项图标统一（去掉了原来只有编辑资料有的那个小图标）。
- **手机端教练设置改成底部弹出菜单**，单手好够；之前是从页面顶部展开。

### 设置 & 账号

- **设置页重组成三大块**：账号 / API / 其他。账号区点邮箱展开二级菜单（改密码 / 退出登录）；API 区分 AI 教练 API + 彩云天气 API；其他放语言 / 默认位置 / 使用手册 / 应用版本。
- **改密码加校验 + 改成弹窗**：先输当前密码再输两次新密码；当前密码错会提示"不正确"+"忘记请联系管理员"。改成居中弹窗 + 背景模糊。桌面端"退出登录"也并进右上角邮箱下拉。
- **彩云天气自带 Token**：设置 → API → 彩云天气 API 可填自己在 dashboard.caiyunapp.com 申请的 token，走自己的配额；留空用共享 token。桌面右上角加了 ☁ 入口。

### 日历 & 天气

- **日历下方 7 天天气卡**：日历格子里的小天气图标/温度去掉，格子更干净；下方新增 7 张天气卡（今天 + 未来 6 天），固定高度两行（第一行星期·日期·图标·温度，第二行体感·湿度·风·AQI）。手机端日历固定在顶部，天气卡可下滑。
- **今日卡修复**：之前"今天"那张温度和体感一直空着，现在显示实时当前温度 + 体感 + 湿度 + 风 + AQI（跟 AI 教练顶部同一份实时数据）。
- **手机端日历格子高度缩小**一截，给天气区让位。
- **7 天天气加"更新于 HH:MM" + ↻ 实时刷新按钮**，点一下绕过缓存重抓。
- 新增[**天气**](weather.md)使用手册章节，讲清每种数据（实时 / 预报 / 训练快照）的更新时机、默认位置、自带 token。

### 赛事 & 录入

- **赛事文案统一**："Half Marathon" → **Half Mar.**，"Marathon" → **Full Mar.**；中文半马 / 全马。
- **斯巴达 tier 上官方色**：Sprint 红 / Super 蓝 / Beast 绿 / **Ultra 紫**。表单选 tier 有色块预览，卡片标签也带色。
- **GAP 输入拆成 分 + 冒号 + 秒** 两个数字框，不用自己敲冒号，秒自动补零。
- 手机端跑步类型 chip 缩短：Easy / Aerobic / Tempo / Interval（桌面端不变）。

### Training 顶部 & 可读性

- **手机端 Training 顶部从三行压成两行**：第一行类型筛选 + Activities/Charts 切换并排，第二行时间范围。电脑端不变。
- 类型筛选改名 **All Types / 全部类型**（本来就是按运动类型筛，紧挨 Activities 切换容易混）。
- **表单长标签不再全大写**（如个人资料里"Race Types You've Done"整句），改正常大小写；活动 tag / 星期 / 组别这些短标记仍大写。
- 手机底部 5 个 tab 的图标和文字调大一档。
- 手机端 Training 顶部时间周期行和统计卡之间的间距修匀（之前比别处宽一截）。

### 切换记忆

- 一次使用中的 sub-tab 选择会**记住**：比如在「比赛」看着"历史"，切去日历再切回来，仍停在"历史"；Training 的 Activities/Charts 选择同理。只有彻底关掉重开 App 才恢复默认。

### 定位整合进个人资料

- 以前定位分两处（个人资料里的"城市"文本 + 设置里的"默认位置"坐标），现在**统一到个人资料的「位置」**一个地方。设置页里的默认位置入口、桌面顶栏的 📍 按钮都去掉了。
- 个人资料 → 位置 点 **📍 用当前位置**：自动定位 + 按 App 语言**自动填好区/市级地址**（如"广州市天河区"）+ 保存坐标。用 GPS 的话一个字都不用打。也可手动输入地区名 + 经纬度兜底。
- 这一个位置同时喂**天气**（坐标）和 **AI 教练**（地址），不再两头维护。

### APP 图标 / 开屏 / 返回键（v0.3.0–v0.3.1，需更新 APK）

- **图标**：外面那圈纯色背景去掉，跟 PWA 一样满幅铺到边，干净整块。
- **开屏**：原来先闪一个 logo、再闪"Training Studio LOADING…"两段式，合并成**一页**（居中 logo + Training Studio）直接进 App。开屏 logo 也裁掉了外圈深色窄框，跟桌面图标一致。安卓 12+ 系统强制的那层开屏背景也统一成米色，过渡更顺。
- **返回键 / 后台常驻**：以前按返回 / 侧滑直接退回桌面、再进要重新开屏（像被杀）。现在：有弹窗先关弹窗 → 不在 Training 页就回 Training 页 → 已在 Training 页则退到后台（App 不被杀，再点秒回，不重开屏）。

### 应用内手册 + 应用内更新（v0.3.2，需更新 APK）

- **用户手册搬进应用**：Guide 入口不再跳外部网页，应用内弹出阅读模式窗口，顶部下拉切章节，离线可看，返回键能关。手机上手册里的表格不再需要左右滑动 —— 宽表自动转成竖排卡片、窄表自动换行铺满。
- **应用内更新（安卓）**：检查到新版后点"立即更新"，应用内下载 APK 并直接拉起安装，不跳浏览器；失败自动回退浏览器下载。首次会让你授权"允许安装未知应用"。

> 图标 / 开屏 / 返回键 / 应用内更新都是安卓原生层改动，需更新到对应 APK 才能体验；网页版刷新即最新。

## 2026-05-28（周四）—— 天气接入

**手动添加训练自动抓天气快照**

- 添加活动时自动抓取当时的天气（温度、体感、湿度、天气状况、风速、AQI），保存进每条训练记录里。
- 表单新增**"开始时间"**字段（可选）：留空就抓保存这一刻的天气；填了就抓那个时间点的历史天气（最近 24 小时）或未来预报。
- 活动卡片紧凑视图行尾显示天气图标 + 温度 + 体感（差 ≥ 2°C 时）。展开后显示完整数据（湿度、风速、AQI）。

**日历**

- 日历每个格子（今天 + 未来 7 天）右上角显示天气图标和温度。未来用预报（最高/最低），今天用实时。
- 点开任意日期的弹窗，最上方有当日天气总结。**过去的日期**显示该天第一条训练录到的天气；**今天和未来**显示实时/预报。

**AI 教练**

- 教练 prompt 新增三块天气数据：当前实时天气、最近 10 条训练每条行尾附当时天气、未来 7 天内的计划训练加预报。
- 教练现在可以基于"今天 32°C 体感 38°C"这种实际数据建议配速和补给。

**默认位置设置**

- 设置页新增**"默认位置"**入口（手机：Settings tab；电脑：顶部右侧📍按钮）。
- 当设备无法提供 GPS（APK 未授权、浏览器拒绝）时，用默认坐标兜底。可以一键"从当前设备获取"。

**Android**

- 安装 `@capacitor/geolocation` 插件，APK 首次抓天气时会弹位置权限请求。
- AndroidManifest 加 `ACCESS_FINE_LOCATION` + `ACCESS_COARSE_LOCATION`。

**小修与发布**

- Training 列表右上角的排序菜单在手机上更紧凑，Distance / Duration 这类选项不会再把下拉框挤出屏幕。
- 准备 v0.2.0 更新包，手机端可通过 APP 内"检查更新"获取新版。
- 修复 Android APP 内天气加载失败：Training 添加记录、日历天气和 AI Coach 顶部天气会正常使用线上天气服务。

> 数据源是**彩云天气** REST API。Token 放在 Vercel 服务端，浏览器看不到。

## 2026-05-27（周三）—— 大杂烩：AI 卡顿修复 + Claude 接入 + 个性化跑步推荐

**UI 操作区打磨**

- 手机底部导航从纯文字改成图标 + 文字，Training / Calendar / Races / AI Coach / Settings 更容易扫。
- 电脑端顶部工具条和主 tab 也同步加上图标，Guide / 语言 / API / 个人资料入口更容易识别。
- Training 的 Add / Upload / Select / Sort 操作集中成带图标的一行，手机端更紧凑。
- AI Coach 对话框上方新增状态 chip，直接显示当前 Provider、教练模式、Memory 状态和日历导入按钮状态。

**AI Coach 回复支持 Markdown 渲染**

- 教练常用的格式（表格、`**粗体**`、`### 标题`、列表、`` `代码` ``、引用块）以前全是字面字符显示，读起来费劲。现在统一用 markdown 引擎渲染。
- 周训练计划那种「7 列大表格」会真正显示成表格；移动端表格如果太宽，可以在气泡内左右**横滑**查看。
- 长 URL / 模型 ID 这类不可断词的内容会自动换行，气泡不会被撑爆。

**AI Coach 体验**

- **发消息不再有 1-2 秒卡顿** —— 用户气泡现在按下发送就出现，不用等数据库写完。
- **每条用户消息下加了「↻ 重新发送」按钮**（只对最近一条显示）。切 app 回来发现报「网络请求失败」时一键重试。
- **网络错误文案改写**，明确告诉你切 app 可能让浏览器暂停了请求 —— 不再让你打开开发者工具看 Network 面板了。
- **移动端聊天气泡变宽**（85% → 94%），长消息不再挤在窄柱里。
- **日历导入按钮可隐藏** —— 在 AI Coach 设置里新增独立「Calendar Import Button」开关（默认显示），不想用就关掉，所有回复下面就不再出现这个按钮。开启时按钮以文字形式显示在每条教练回复下方。

**Claude API 回来了（第三方中转）+ 模型锁定为最强**

- API 设置加了 **Provider 切换器**：DeepSeek 和 Claude 都能用，两个 Key 同时存着，随时切换不需要重新粘。设置弹窗只显示当前 provider 的设置，UI 不再混在一起。
- Claude API 走的是**第三方中转服务**（console.claudeapi.com），不是 Anthropic 官方。提供 4 条接入线路（默认 / 香港 / 日本 / 新加坡），按当前网络选一条，按设备保存。
- **模型锁定为该 provider 的最强款**（DeepSeek = v4-pro，Claude = opus-4-7）。用户不再需要选 —— 厂商发布更强模型时由开发者统一更新，应用下次启动自动跟进。
- 设置弹窗顶部增加 **价格对比卡片** —— 列出两家的输入/输出价格、单条消息估算成本、倍数差。Guide 文档同步加了价格表。可以一眼看出 Claude 单条约 ¥0.20、DeepSeek 不到 2 分钱。
- 切换 Provider 只影响下一条消息发给谁，过去的对话历史保留。

**跑步类型推荐：从固定阈值 → 个性化心率区间**

- 录入训练时，填好平均心率后，系统会按你**个人的 5 区心率**推荐配速子类（Easy / Aerobic / Tempo），匹配的 chip 出现「推荐」标记 + 绿色描边。
- 推荐只是建议，你想选别的就直接点别的，系统不会替你做决定。
- Z4+ 不主动推荐 —— 平均心率分不清是 Tempo 还是 Interval，交给你手动选。
- 没填 Resting HR / Max HR 的用户会退回到旧的统一阈值。
- 同样的逻辑也用于 Garmin CSV 导入的自动预填。

**Training stats 重排 + 时间显示**

- 顶部统计卡片从 **Sessions / Distance / Ascent / Avg HR** 改成 **Sessions / Time / Distance / Ascent**。平均心率作为汇总指标参考价值不大，换成总时长更有用。
- 移动端的 Time 只显示「Xh」/「Xm」格式，避免换行。

**Charts 简化**

- 默认时间窗口从 8 周改为 4 周。
- 删除「心率区间分布图」—— 单凭每次的平均心率分桶低估了多样性，留着反而误导。
- 力量训练时长按部位**比例分配**：选中「Upper Body」筛选时，一次「上肢+下肢」30 分钟的训练在该图里只计 15 分钟。

## 2026-05-20（周六）—— 数据搬家日：从浏览器本地搬到云端

整整一天搬家。在此之前所有数据只存在你打开过的那个浏览器里 —— 换设备就丢。当天之后每个账号自己的数据上云，换设备能继续用。
